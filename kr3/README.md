# Приложение для заметок — практические работы 13–18

Репозиторий содержит реализацию прогрессивного веб-приложения (PWA) для управления заметками с постепенным усложнением функциональности: от базового офлайн-доступа через Service Worker до системы напоминаний с WebSocket и системными уведомлениями.

## Структура проекта

```
kr3/
├── content/
│   ├── home.html           # Динамический контент главной страницы (App Shell)
│   └── about.html          # Страница «О приложении»
├── icons/                  # Набор иконок разных размеров
├── server/
│   └── server.js           # Node.js сервер (Express + Socket.IO + web-push)
├── index.html              # Каркас приложения (App Shell)
├── app.js                  # Основная логика клиента
├── sw.js                   # Service Worker
├── manifest.json           # Web App Manifest
├── package.json
├── localhost+2.pem         # TLS-сертификат для локального HTTPS
└── localhost+2-key.pem     # Приватный ключ для локального HTTPS
```

## Практическое задание 13 — Service Worker

Разработано базовое веб-приложение для управления заметками с поддержкой офлайн-режима через Service Worker. Данные хранятся в `localStorage`, а статические ресурсы кэшируются при первой загрузке.

**Технологии:** HTML, CSS, JavaScript, Service Worker API, localStorage

### Реализованный функционал

- Просмотр и добавление заметок.
- Сохранение данных в `localStorage`.
- Регистрация Service Worker и кэширование статических ресурсов (`index.html`, `app.js`, `manifest.json`, иконки).
- При отсутствии сети приложение загружается из кэша и остается работоспособным.

### Жизненный цикл Service Worker

| Событие    | Действие                                             |
| ---------- | ---------------------------------------------------- |
| `install`  | Предварительное кэширование статических файлов       |
| `activate` | Очистка устаревших кэшей                             |
| `fetch`    | Возврат ресурса из кэша или из сети для `/content/*` |

## Практическое задание 15 — HTTPS + App Shell

Настроено локальное приложение в стиле App Shell с динамическим контентом, загружаемым без перезагрузки.

**Технологии:** App Shell, локальное HTTPS, fetch API

### Реализованный функционал

- Архитектура App Shell: основной каркас приложения находится в `index.html`.
- Динамический контент вынесен в `content/home.html` и `content/about.html`.
- Навигация выполнена через JavaScript и `fetch` без перезагрузки страницы.
- Добавлена страница «О приложении».
- В проекте присутствуют локальные TLS-сертификаты `localhost+2.pem` и `localhost+2-key.pem`.

### Стратегии кэширования

| Тип ресурса                         | Стратегия                       |
| ----------------------------------- | ------------------------------- |
| Статика (App Shell)                 | Cache First                     |
| Динамический контент (`/content/*`) | Network First с fallback на кэш |

## Практическое задание 16 — WebSocket + Push

Добавлена двухсторонняя связь в реальном времени через Socket.IO и поддержка уведомлений с использованием Notification API.

**Технологии:** Node.js, Express, Socket.IO, web-push, Notification API

### Реализованный функционал

- Создан Node.js-сервер на `server/server.js` с Express и Socket.IO.
- Клиент отправляет событие `newTask` при добавлении заметки.
- Сервер рассылает `taskAdded` всем подключённым клиентам.
- Реализована кнопка управления уведомлениями: "Включить уведомления" / "Отключить уведомления".
- Система уведомлений построена на клиентских уведомлениях через Service Worker и `showNotification`.

### API эндпоинты

| Метод  | URL            | Описание                 |
| ------ | -------------- | ------------------------ |
| `POST` | `/subscribe`   | Сохранение push-подписки |
| `POST` | `/unsubscribe` | Удаление push-подписки   |

### WebSocket события

| Событие     | Направление          | Описание                  |
| ----------- | -------------------- | ------------------------- |
| `newTask`   | Клиент → Сервер      | Новая заметка добавлена   |
| `taskAdded` | Сервер → Все клиенты | Оповещение о новой задаче |

## Практическое задание 17 — Система напоминаний

Реализована система напоминаний с возможностью отложить событие на 5 минут.

**Технологии:** Node.js, setTimeout, WebSocket, Notification Actions API

### Реализованный функционал

- На форме добавлено поле `datetime-local` для времени напоминания.
- Заметки в `localStorage` хранят `id`, `text` и `reminder` (timestamp).
- При создании заметки с напоминанием клиент отправляет событие `newReminder` на сервер.
- Сервер планирует напоминание через `setTimeout` и хранит активные таймеры в `Map`.
- При срабатывании напоминания сервер отправляет событие `reminderTriggered` всем клиентам.
- Системное уведомление содержит действие: "Отложить на 5 минут".
- При нажатии на действие уведомления Service Worker делает `POST /snooze`.
- Сервер отменяет старый таймер и создаёт новый на 5 минут.

### API эндпоинты

| Метод  | URL                       | Описание                            |
| ------ | ------------------------- | ----------------------------------- |
| `POST` | `/subscribe`              | Сохранение push-подписки            |
| `POST` | `/unsubscribe`            | Удаление push-подписки              |
| `POST` | `/snooze?reminderId={id}` | Откладывание напоминания на 5 минут |

### WebSocket события (обновлённые)

| Событие             | Направление          | Описание                         |
| ------------------- | -------------------- | -------------------------------- |
| `newTask`           | Клиент → Сервер      | Обычная заметка добавлена        |
| `taskAdded`         | Сервер → Все клиенты | Уведомление о новой задаче       |
| `newReminder`       | Клиент → Сервер      | Заметка с напоминанием добавлена |
| `reminderTriggered` | Сервер → Все клиенты | Сработавшее напоминание          |

## Особенности реализации

- Основной код клиента находится в `app.js`.
- Каркас приложения загружается из `index.html`, а контент — из файлов в `content/`.
